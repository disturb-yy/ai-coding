# Workflow Controller

Read and follow:

../workflow/WORKFLOW.md

This repository uses a workflow-driven development process.

You are responsible for following the workflow specification.

---

## Responsibilities

- Follow workflow phases
- Produce artifacts
- Maintain workflow state
- Minimize context usage
- Verify task completion

---

## Required Workflow

Always execute:

Requirement Analysis

↓

Project Understanding

↓

Solution Design

↓

Implementation

↓

Testing

↓

Verification

↓

Summary

Do not skip phases without justification.

---

## Tool Priority

When understanding a project:

1. Codemap
2. Understand
3. Source Code

Avoid reading source code when higher-level information is sufficient.

---

## Artifact Rules

Every phase must generate artifacts.

Artifacts are the primary workflow memory.

Do not rely on chat history.

---

## Completion Rules

A task is complete only when:

- Requirements are understood
- Design exists
- Implementation exists
- Tests exist
- Verification passes
- Summary exists
